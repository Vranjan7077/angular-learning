Angular Routing Demo

This project demonstrates how routing works in Angular using simple live examples for static routes, dynamic routes, nested routes, lazy loading, and wildcard routing.

Step-by-step demo flow

  1. Start on /home and read the demo cards.
  2. Open /users to see a static route in action.
  3. Open any user to move to /users/:id.
  4. Open the Posts tab to see the nested route.
  5. Click any post to open /users/:id/posts/:postId.
  6. Try /demo-not-found to trigger the wildcard route.

Where this is implemented

  • app.routes.ts defines the routes
  • user-detail-page.component.ts reads route params
  • user-detail-page.component.html renders nested routes
  • user-posts-page.component.ts loads posts for a user
  • user-post-detail-page.component.ts loads one selected post

Dynamic Route

The user detail page reads the id from the route and loads data for that user.

Example: /users/1

Observe: the id value changes in the URL.

Implemented in: ActivatedRoute

Nested Route

The posts page is shown inside the user detail page using a nested router outlet.

Example: /users/1/posts

Observe: posts render inside the detail page layout.

Implemented in: router-outlet

Open Nested Route

Post Detail Route

Clicking a post opens a more specific route for that selected post under the same user.

Example: /users/1/posts/1

Observe: the URL now includes both user id and post id.

Implemented in: user-post-detail-page.component.ts

Open Post 1

Lazy Loading

The posts page is loaded only when you visit the posts route instead of loading it upfront.

Used in: loadComponent()

Observe: the posts view loads only when opened.

Implemented in: app.routes.ts

Wildcard Route

Unknown routes are handled by the wildcard route and redirected to the not-found page.

Example: /demo-not-found

Observe: unknown URLs go to the 404 page.

Implemented in: path: '**'

Route Params + API

Route parameters are combined with API calls to fetch the correct user and posts data.

Uses: ActivatedRoute + HttpClient

Observe: different ids load different users.

What to observe

  • Notice how the URL changes as you move between pages.
  • See how :id changes which user is loaded.
  • Observe that the posts page appears inside the detail page.
  • Click a post and observe the deeper post detail route.
  • Try a missing route to trigger the 404 page.

Core Angular Router APIs

  • routerLink for navigation
  • Routes for route definitions
  • ActivatedRoute for route params
  • router-outlet for rendering route content
  • loadComponent for lazy loading
